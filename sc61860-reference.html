HITACHI SC61860 • 8-BIT CPU • USED IN SHARP PC-12xx / 13xx / 14xx

About

The SC61860 is an 8-bit CPU developed by Hitachi, using the ESR-H architecture. Its most prominent use was in various pocket computers and organizers produced by Sharp in the 1980s, notably the PC-12xx, PC-13xx, and PC-14xx series.

Despite its age, various details about this CPU remain shrouded by obscurity. The original instruction set documentation is based on community research compiled by utz82, published under the CC0-1.0 license.

Architecture

  • Big-endian 16-bit address encoding in instructions
  • No interrupts
  • 8-bit data bus
  • 64 KB address space
  • 8 KB of integrated ROM at 0x0000вҖ“0x1FFF, accessible only via the DATA instruction
  • 96 bytes of scratchpad memory (internal RAM) вҖ” used for registers, stack, I/O, and BCD calculations

The system stack starts at internal RAM address 0x5B and grows downwards towards 0x0C. The I/O ports IA, IB, and F0 are mapped to internal RAM addresses 0x5BвҖ“0x5E.

Internal RAM registers 0x30вҖ“0x3F are used by the system and BASIC interpreter вҖ” these should not be modified.

All numeric values in this reference are hexadecimal unless stated otherwise.

Using the LOOP Instruction

The LOOP instruction allows execution of conditional loops without consuming a general-purpose register. It is also 1 cycle faster than the equivalent DECr + JRNZM pattern.

        LIA     n       ; n = [number of iterations] - 1
        PUSH            ; store counter in (R)
lloop:
        ...             ; loop body
        LOOP    lloop   ; (R)-1вҶ’(R), branch back if carryвү 0

The loop can be broken early by executing LEAVE, which sets (R) to 0.

Using PTC / DTC (Switch-Case)

The PTC/DTC instruction pair (also known as CASE1/CASE2, PTJ/DTJ, CPCAL/DTLRA, SETT/JST) implements hardware switch-case. Despite its common name "Prepare/Do Table Jump", it does not jump вҖ” it pushes a return address and then calls a subroutine.

Handling varies between assemblers. In most, the arguments are .db/.dw data statements following the opcode:

        PTC
        .db     k       ; k = number of cases
        .dw     nm      ; nm = return address after all subroutines

        DTC
        .db     n       ; n = condition value matched against A
        .dw     nm      ; nm = call address for this case
        ...             ; repeat k times
        .dw     nm      ; nm = default/fallthrough call address
